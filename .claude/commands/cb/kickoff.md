## Phase 0 절차 (순서 고정)

1. **프로파일 인터뷰** (§4.5.2 — 스킬 후보 질문 병합)
   사람에게 아래 8개 항목을 한 번에 묻는다 (이미 답이 나온 항목은 건너뜀):
   1. 기술 스택 (백엔드/프론트엔드/DB/빌드 도구 + 버전)
   2. 환경 제약 (폐쇄망 여부, 프록시, 사용 가능 레지스트리/미러, 배포 대상)
   3. 코딩 규약 (네이밍, 패키지 구조, 린터/포매터, 기존 코드베이스 유무)
   4. 테스트 전략 (프레임워크, 커버리지 기대치, E2E 도구, 실행 명령)
   5. Git/브랜치 전략 (브랜치 명명, 커밋 규약, PR 정책)
   6. 산출물 요구 (고객 제출용 산출물 양식 — SI 표준 산출물 필요 여부)
   7. 품질/보안 요구 수준 (인증·인가 방식, 감사 로그, 개인정보 취급 여부)
   8. 일정/마일스톤 골격
   + 스택 스킬 후보 도출·부족분 질문 (§4.5.5 의 1~3단계 병합)

2. **project-profile.md 작성** (docs/00-charter/project-profile.md)
   §4.5.3 템플릿 기준. 8절에 활성 스택 스킬 목록 포함.
   → **[사람 게이트]** 프로파일 승인·커밋

3. **스택 스킬 생성** + settings.json 스택 명령 교체 (§4.5.5 의 5~6단계)

4. **프로젝트 헌장 초안** (docs/00-charter/charter.md)
   목표/범위/제약/마일스톤 골격/Done의 정의
   → **[사람 게이트]** 헌장 승인

5. **인테이크 모드 판정** (§4.6.1 — 소형이면 대화형, 기본은 템플릿형)
   - 템플릿형: `docs/10-requirements/intake/_TEMPLATE.md` 를 복사해
     프로젝트 루트에 `INTAKE-<영역>.md` 발급
     → **"작성을 마치면 `/cb:intake` 를 실행하세요"** 안내 후 킥오프 일시 정지
   - 대화형: PM 이 템플릿 항목을 대화로 질의해 인테이크 파일을 대신 작성 → 확인

6. **마일스톤·트래킹 이슈 골격 생성** (백로그는 Phase 1 완료 후)
