인자: $ARGUMENTS (마일스톤 이름 또는 번호)

docs/GUIDE.md 의 §11 자기개선 루프 절차를 따른다.

## 절차

1. **회고 초안 작성**
   마일스톤의 전체 이슈/코멘트/REWORK 사유를 훑고:
   - 잘된 것
   - 문제와 원인
   - 액션 아이템
   → `docs/90-retro/RETRO-<MS>.md` 로 저장

2. **반복 문제·검증된 해법 패턴 추출**
   "반복될 문제 + 검증된 해법" 패턴을 식별한다.
   예: "환경 문제로 빌드 실패 시 복구 절차", "특정 라이브러리 호환성 이슈"

3. **learned 스킬 초안 생성**
   각 패턴을 `.claude/skills/learned/<slug>/SKILL.md` 로 생성.
   양식:
   ```
   ---
   name: learned-<slug>
   description: >
     <어떤 상황에서 발동해야 하는지 — 구체적 트리거>.
     <프로젝트명> #<이슈> 에서 검증됨.
   ---
   ## 문제
   ## 검증된 해법 (절차)
   ## 주의점
   ## 증거: 이슈 #n, 커밋 <hash>
   ```

4. **에이전트 정의 개선 제안**
   반복된 REWORK 사유 → 해당 에이전트 품질 기준에 추가할 내용 목록 제시

5. **[사람 게이트]** learned 스킬 목록을 사람이 검토·승인 후 커밋
   (오염 방지 — 잘못된 교훈 축적 방지)

6. **다음 마일스톤 진입 안내**
   ```
   ✅ [마일스톤명] 마감 완료. 다음 중 하나를 선택해주세요:
   • 회고 먼저 진행 → /cb:retro (이미 완료)
   • 다음 마일스톤 시작 → /cb:plan-sprint <다음MS>
   ```
