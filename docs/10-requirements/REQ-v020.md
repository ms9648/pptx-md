## 1. 전제·불변식 (모든 신규 FR 공통) — REQ-wave3 §0 계승

- **INV-1 결정성(ADR-218 계승)**: **텍스트 경로**는 동일 IR → 동일 Markdown 을 유지한다.
  렌더→VLM 경로 출력은 본질적으로 비결정적이므로 INV-1 의 "바이트 동일" 대상에서 제외하되,
  **경로 라우팅 판정(FR-31)과 슬라이드 결합 순서(FR-33)는 결정적**이어야 한다(동일 IR →
  동일 라우팅 집합·동일 배치). 비결정 구간은 VLM 응답 본문에 국한된다.
- **INV-2 IR 하위호환(ADR-202/205)**: `ir.py` 필드 추가 시 항상 default 유지, 기존 호출·테스트 무손상.
- **INV-3 옵션 하위호환(ADR-602)**: 신규 `ConvertOptions` 필드는 frozen dataclass 에 default 추가.
  **신규 동작은 전부 opt-in**(기본값 = 현행 출력 동등). `ConvertOptions()`·`convert(src)` 무손상.
  (Wave3 전례: `heading_hierarchy` 등 4개 옵션 default `False` 준수)
- **INV-4 격리(ADR-204/214 계승)**: 슬라이드/도형/렌더/VLM콜 1건 실패가 전체 변환을 중단시키지
  않는다. 렌더 실패·VLM 실패 슬라이드는 **텍스트 경로로 폴백**한다(무손실 원칙).
- **INV-5 의존 격리(NFR-08 계승)**: `assembler.py`·`validator.py` 최상단 VLM SDK·python-pptx·Pillow
  import 0 유지. 렌더/VLM 모듈은 별도 내부 모듈로 분리하고 `describer=None`·렌더 미가용 시
  core-only 환경에서 import·convert 가 성공해야 한다.
- **INV-6 프로파일 준수(§1)**: 신규 외부 파이썬 라이브러리 도입은 **임의 확정 금지**. 슬라이드
  래스터화 렌더 엔진은 LibreOffice(외부 실행파일, py 의존 아님)로 확정되었으나, PDF→PNG 변환에
  신규 **파이썬** 의존이 필요하면 §8 `DEC-7`(NEEDS_DECISION, ARCH 위임)로만 승격한다.

## 2. FR-30 슬라이드→이미지 렌더링 (R-A1)

**스토리**: 파이프라인 개발자로서, 시각 복잡 슬라이드 1장을 원본 레이아웃 그대로 PNG 로
렌더링해 VLM 입력으로 쓰고 싶다. 임베디드 이미지가 0장이어도(현 VLM 경로 무력) 2D 배치를
보존한 시각 입력을 확보하기 위해서다.

**렌더 엔진 = LibreOffice 확정(DEC-1)**: `soffice` 로 PPTX → **PDF** 변환 후, PDF 페이지를
슬라이드별 PNG 로 래스터화하는 경로를 채택한다. `soffice --convert-to png` 단독은 멀티슬라이드
PPTX 에서 1번 슬라이드만 내보내는 제약이 있어 **PDF 경유**로 해소한다. COM(Windows 전용) 경로는
**기각**. PDF→PNG 래스터화에 쓰이는 구체 파이썬 라이브러리(⟨ARCH⟩)와 목표 DPI/해상도·타임아웃
수치는 **architect ADR 위임**이며 프로파일 §1 신규 의존 승인 게이트(DEC-7)를 유지한다.

**AC(베이스라인)**:
- [ ] AC1: Given 유효한 PPTX 와 슬라이드 인덱스 N When `render_slide(pptx_path, n)` 호출
      Then PNG 바이트(`b"\x89PNG"` 시작)를 반환하거나, 렌더 미가용 시 `None` 을 반환한다
      (예외 미전파 — INV-4, `vector.convert_vector_to_png` 의 graceful-skip 규약 계승).
- [ ] AC2: Given 렌더 엔진 미설치(예: LibreOffice `soffice` PATH 부재) When 렌더 호출
      Then `None` 을 반환하고 경고 로그 1건을 남기며 프로세스는 계속된다(폴백은 FR-33 소관).
- [ ] AC3: Given 64슬라이드 실증 파일의 임의 슬라이드 When 렌더(soffice→PDF→PNG)
      Then 반환 PNG 는 최소 폭·높이 `⟨ARCH: 목표 DPI/해상도⟩` 이상이고 슬라이드 종횡비를
      보존한다(`PresentationIR.slide_width_emu/height_emu` 비율 ±1% 이내).
- [ ] AC4 (경계): Given 존재하지 않는 슬라이드 인덱스(음수·범위 초과) When 렌더
      Then `None` 반환(예외 미전파). Given 손상 PPTX When 렌더 Then `None` 반환.
- [ ] AC5 (경계): Given 동일 PPTX·동일 슬라이드 2회 렌더 When 비교
      Then 렌더러가 결정적 산출을 보장하는 범위에서 PNG 는 재현 가능해야 한다
      (LibreOffice 특성상 바이트 동일이 불가하면 `⟨DEC-6⟩` 회귀 전략이 이를 흡수 — 여기서는 비강제).
- [ ] AC6 (성능·비용 연계): 렌더 서브프로세스(soffice)는 타임아웃 상한(`⟨ARCH: 예 30s⟩`)을 가지며
      초과 시 `None` 반환(`vector.VECTOR_TIMEOUT_S` 규약 계승). NFR-V020-1 참조.

**설계 노트(요구 아님, ARCH 입력)**: 기존 `src/pptx_md/vector.py` 는 이미 LibreOffice
`--convert-to png`(EMF/WMF 단건)에 사용 중이므로 `soffice` 배선은 재사용 가능하다. 멀티슬라이드
PPTX 는 `--convert-to pdf` 후 PDF 페이지 래스터화로 슬라이드별 PNG 를 확보한다. **PDF→PNG
래스터화 파이썬 라이브러리 선정은 `DEC-7`(신규 의존, ARCH ADR + 사람 승인)** 로 남는다.

**우선순위**: Must / **의존**: FR-33(호출 주체), (재사용) `vector.py` LibreOffice(`soffice`) 배선

## 3. FR-31 시각 복잡도 탐지 (R-A2)

**스토리**: 오케스트레이터로서, 어떤 슬라이드를 (비싼) 렌더+VLM 경로로 보낼지 IR 만으로
결정적으로 선별하고 싶다. 텍스트 슬라이드에 불필요한 VLM 비용이 나가지 않게 하고, S5/S7/S12
같은 시각 구조 슬라이드는 놓치지 않기 위해서다.

> **VLM 정책 = 품질 우선(DEC-3/5)**: 시각 슬라이드를 **넓게 커버**하는 것을 지향한다. 복잡도
> 임계는 재현율(놓치지 않음) 쪽으로 기운다. 단, 구체 임계·가중치 수치는 `⟨DEC-2⟩`(ARCH 위임).

**AC(베이스라인)**:
- [ ] AC1: Given `SlideIR` When `is_visually_complex(slide) -> bool` 판정
      Then 판정은 **IR 파생 특징만** 사용(도형 수, 커넥터/`OtherShapeIR` 수, 차트/SmartArt
      GraphicFrame 존재, `GroupShapeIR` 존재, 좌표 기반 열/행 정렬도, 텍스트 선형성)하며
      **VLM·렌더 호출 없이** 결정적으로 반환한다(동일 IR → 동일 bool, INV-1).
- [ ] AC2: Given 좌표(EMU)상 3개 이상 텍스트 도형이 유사 top 밴드에 정렬되어 다열 구조를
      형성하는 슬라이드(S5 "4열 매핑" 류) When 판정 Then `True`(멀티컬럼 신호).
- [ ] AC3: Given 커넥터/화살표(`OtherShapeIR` 또는 도형 흐름) 또는 차트(FR-26 GraphicFrame)가
      `⟨DEC-2: 임계 N⟩` 이상 존재하는 슬라이드(S7 흐름 / S12 차트 류) When 판정 Then `True`.
- [ ] AC4 (정상-부정): Given 제목 + 단일 텍스트 프레임의 선형 불릿 슬라이드 When 판정
      Then `False`(텍스트 경로 유지). 순수 텍스트 슬라이드는 렌더 경로로 새지 않는다.
- [ ] AC5 (경계): Given 도형 0개(빈 슬라이드) When 판정 Then `False`.
- [ ] AC6 (튜닝 가능성): 판정 임계값은 **명명 상수**로 노출하고(classifier.py 임계 상수 전례)
      `⟨DEC-2: 복잡도 기준·가중치⟩` 확정 시 값이 고정된다. 임계는 실증 파일 S5/S7/S12 를
      `True`, 순수 텍스트 슬라이드를 `False` 로 판정하는 것을 캘리브레이션 목표로 하며,
      품질 우선(DEC-3/5) 정책상 경계 시 렌더 경로로 기운다(과소탐지보다 과탐지 허용).
- [ ] AC7 (오버라이드): Given 사용자가 슬라이드 인덱스 강제 목록(`force_render`/`force_text`)을
      옵션으로 제공 When 판정 Then 휴리스틱보다 사용자 지정이 우선한다(비용/정확도 수동 제어).
      *(옵션 표면은 FR-34 와 공유 — `⟨DEC-4⟩` 확정 대상)*

**우선순위**: Must / **의존**: FR-23(좌표 EMU, 기존), FR-26(차트/SmartArt 추출, 기존), FR-33(소비 주체)

## 4. FR-32 VLM 슬라이드 재구성 (R-A3)

**스토리**: RAG 파이프라인 빌더로서, 렌더된 슬라이드 이미지를 VLM 이 통째로 읽고
표/중첩리스트/흐름/차트데이터를 구조화 Markdown 으로 재구성해 주기를 원한다. S5 4열 매핑,
S7 AS-IS↔TO-BE 행 대응, S12 KPI↔라벨·도넛 %↔카테고리의 관계를 보존하기 위해서다.

> **VLM 정책 = 품질 우선(DEC-3/5) 확정**: 고성능 모델 사용을 지향하고 시각 슬라이드를 넓게
> 커버한다. 구체 **모델명·해상도·토큰 상한·콜 상한 수치는 architect 제안(⟨DEC-3/5⟩)** 으로
> 위임하며, 프로파일 §1 `[vlm]` extras 범위 내 SDK 선택을 유지한다.

**AC(베이스라인)**:
- [ ] AC1: Given 렌더 PNG + 슬라이드 컨텍스트(제목/텍스트 IR) When 슬라이드 재구성 프로바이더
      호출 Then 유효한 Markdown 조각(표는 정합 pipe table, 중첩 리스트는 들여쓰기 계층, 흐름은
      순서 표현)을 반환한다. 반환물은 FR-14/FR-29 validator 경고 0 을 목표로 한다(깨진 표·중복
      헤딩·전공백 표 0).
- [ ] AC2: Given 프로바이더가 표 구조를 담은 슬라이드 When 재구성 Then 헤더 행 + 구분 행 +
      1개 이상 데이터 행을 갖는 pipe table 이 생성된다(FR-29 AC1 "데이터 행 0" 위반 회피).
- [ ] AC3 (프로토콜 관계): 슬라이드 재구성 계약은 이미지 단위 `ImageDescriber.describe(
      image_bytes, image_ext, shape_hint) -> str` 과 **입력 컨텍스트·출력 단위가 다르므로**,
      기존 프로토콜 재사용 여부 또는 신규 프로토콜(`SlideReconstructor` 류) 도입은
      `⟨DEC-8⟩` 로 확정한다. 어떤 선택이든 `assembler`/`validator` 는 VLM SDK 를 import 하지
      않는다(INV-5).
- [ ] AC4 (프롬프트 전략): 재구성 프롬프트는 "표/중첩리스트/흐름/차트 데이터를 우선 구조화,
      불확실하면 원문 텍스트 보존" 정책을 명시하고, 출력 형식(순수 Markdown, 코드펜스 금지 등)을
      제약한다. 프롬프트 문구·모델·해상도·토큰 상한은 `⟨DEC-3⟩`(품질 우선, ARCH 제안) 확정 대상.
- [ ] AC5 (격리): Given VLM 이 실패/타임아웃/빈 응답 When 재구성 Then 예외를 전파하지 않고
      해당 슬라이드는 텍스트 경로 산출로 폴백한다(무손실, INV-4). 실패는 경고 로그(원문 텍스트
      미출력, NFR-06) 1건.
- [ ] AC6 (경계): Given VLM 응답이 구조화에 실패한 산문 When 재구성 Then 원문 텍스트를 그대로
      담아 콘텐츠 손실 0(mermaid fallback 전례 계승). 깨진 부분 표는 넣지 않는다.
- [ ] AC7 (PII): 렌더 PNG·VLM 프롬프트·응답 로깅 시 원본 텍스트/PII 를 로그에 출력하지 않는다
      (프로파일 §7, NFR-06). 마스킹 옵션(FR-15) 활성 시 재구성 산출에도 마스킹이 적용되는지는
      `⟨DEC-3⟩` 하위 항목으로 확정.

**우선순위**: Must / **의존**: FR-30(렌더 입력), FR-33(호출 주체), FR-08(ImageDescriber 프로토콜, 관계 정의), FR-29(validator 품질 기준)

## 5. FR-33 하이브리드 오케스트레이션 (R-A4)

**스토리**: 라이브러리 사용자로서, 한 번의 `convert()` 호출로 텍스트 슬라이드는 저비용 추출,
시각 복잡 슬라이드는 렌더→VLM 재구성을 자동 적용하고, 슬라이드 순서대로 하나의 Markdown
문서로 결합받고 싶다.

**AC(베이스라인)**:
- [ ] AC1: Given `ConvertOptions` 신규 옵션 `visual_reconstruct=True`(가칭)와 슬라이드
      재구성 프로바이더 주입 When `convert()` Then FR-31 이 `True` 로 판정한 슬라이드만
      렌더→VLM 경로, 나머지는 현행 텍스트 경로로 처리되어 **슬라이드 인덱스 순**으로 병합된다
      (ADR-220 병합 순서 계승, 결정적 배치).
- [ ] AC2 (하위호환): Given 신규 옵션 미지정(기본) When `convert(src)` Then 출력은 현행과
      **바이트 동일**(전 신규 동작 opt-in, INV-3). 회귀 diff 0.
- [ ] AC3 (프로바이더 부재): Given `visual_reconstruct=True` 이나 재구성 프로바이더 `None`
      또는 렌더 미가용 When `convert()` Then 렌더 경로가 무력화되고 전 슬라이드가 텍스트
      경로로 처리된다(예외 없음, core-only 환경 안전, INV-4/INV-5).
- [ ] AC4 (결합 규약): Given 혼합 문서 When 어셈블 Then 렌더 경로 슬라이드도 텍스트 경로와
      동일한 슬라이드 블록 규약(`## Slide N`/헤딩/구분자)을 따르고, FR-28 구조화 옵션과
      상호 파손 없이 결합된다(구분자 `---` 충돌 0).
- [ ] AC5 (격리): Given 한 슬라이드의 렌더/VLM 실패 When `convert()` Then 그 슬라이드만 텍스트
      폴백되고 다른 슬라이드·전체 문서는 완료된다(INV-4). 부분 실패 문서도 유효 Markdown.
- [ ] AC6 (관측): 변환 종료 시 렌더 경로 슬라이드 수·VLM 콜 수·폴백 수를 로그 메타로 남긴다
      (원문 미출력, NFR-06). 비용 상한 계측(FR-34)의 입력이 된다.

**우선순위**: Must / **의존**: FR-30, FR-31, FR-32, FR-34(비용 상한), FR-16(convert 오케스트레이터, 기존), FR-28(결합 규약)

## 6. FR-34 이미지 임베드 옵션 & 비용 제어 (R-A5)

**스토리**: 비용에 민감한 사용자로서, 렌더 이미지 병행 삽입 여부와 문서당 VLM 콜 상한을
제어하고, 동일 슬라이드 재변환 시 캐시로 콜을 아끼고 싶다.

> **비용 vs 품질 균형**: v0.2.0 방향은 **품질 우선(DEC-3/5)** 이므로 콜 상한 기본값은 넓은
> 커버리지를 지향한다. 단 구체 `max_vlm_slides` 기본 수치·선정 규칙은 `⟨DEC-3/5⟩`(ARCH 제안)
> 확정 대상이다. 비용 제어 옵션 표면 자체는 요구로 확정(제공한다).

**AC(베이스라인)**:
- [ ] AC1 (임베드 옵션): Given `embed_rendered_image` 옵션 When 렌더 경로 슬라이드 어셈블
      Then 옵션 활성 시 렌더 PNG 를 (저장/참조 방식 `⟨DEC-4⟩`) Markdown 에 병행 삽입하고,
      기본값(`⟨DEC-4: 권장 False⟩`)에서는 재구성 텍스트만 출력한다(opt-in, INV-3).
- [ ] AC2 (콜 상한): Given `max_vlm_slides`(가칭, 문서당 렌더+VLM 슬라이드 상한) When 복잡
      슬라이드 수가 상한 초과 Then 상한까지만 렌더 경로, 초과분은 텍스트 폴백하고 경고 로그를
      남긴다. 상한 기본값·선정 규칙(복잡도 상위 N)은 `⟨DEC-3/DEC-5⟩`(품질 우선, ARCH 제안) 확정.
- [ ] AC3 (캐싱): Given 동일 렌더 PNG 가 재등장(sha256 동일) When 재구성 Then 재구성 결과를
      1회만 산출하고 재사용한다(FR-27 `_group_by_hash` sha256 캐시 전례 확장). 캐시 수명은
      호출 범위 한정(디스크/세션 캐시는 §7 Non-goal, REQ P-01 별도 승인 영역).
- [ ] AC4 (경계): Given `max_vlm_slides=0` When `convert()` Then 렌더 경로 0건(전 텍스트),
      VLM 콜 0. Given 상한 미지정 When 처리 Then `⟨DEC-5⟩` 기본 정책 적용.
- [ ] AC5 (콜 수 계측): 문서 변환의 실제 VLM 콜 수가 (고유 복잡 슬라이드 수, 상한) 안에
      있음을 계측하는 테스트가 가능해야 한다(NFR-V020-2). 중복 렌더 캐시 히트는 콜에서 제외.

**우선순위**: Should / **의존**: FR-33(경로 수), FR-32(재구성 콜), FR-27(sha256 캐시 전례), FR-30(렌더 산출 임베드)

## 7. FR-35 회귀 검증 전략 — 비결정 VLM 골든 게이트 (R-A6)

**스토리**: CI/유지보수자로서, VLM 출력이 비결정적이어도 하이브리드 경로의 회귀(라우팅 오류,
결합 파손, 텍스트 경로 퇴행)를 자동으로 잡아내고, 기존 FR-29 골든 게이트를 깨지 않기를 원한다.

> **픽스처 정책(RISK 반영)**: 실증 파일 `.upload/SamsungSDS_AICC_제안서.pptx` 는 **사내 자료**
> 이므로 PyPI 패키지·공개 CI 로 배포되는 골든 픽스처로 **직접 사용 불가**하다. 회귀 게이트의
> 공개 픽스처는 S5/S7/S12 구조 결함(4열 매핑/AS-IS↔TO-BE/KPI·도넛)을 재현하는 **합성 PPTX
> 픽스처**로 대체한다. 실증 파일은 로컬 캘리브레이션·수동 검증에만 쓰고 리포지토리·배포 산출에
> 포함하지 않는다.

**AC(베이스라인)**:
- [ ] AC1 (텍스트 경로 골든 불변): Given 신규 옵션 전부 off When 기존 off/on 골든(FR-29 AC8
      2종) 변환 Then diff 0 으로 통과한다. v0.2.0 배선이 기존 골든을 회귀시키지 않는다(INV-3).
- [ ] AC2 (라우팅 결정성 골든): Given **합성 픽스처**(S5/S7/S12 재현 + 순수 텍스트) When FR-31
      판정 Then "렌더 경로 슬라이드 인덱스 집합"이 결정적 골든과 일치한다(VLM 미호출, 순수 IR
      판정 — 결정적으로 계측 가능). 사내 실증 파일은 공개 골든에 미포함.
- [ ] AC3 (모의 프로바이더 골든): Given 결정적 **모의 재구성 프로바이더**(고정 응답) 주입
      When `visual_reconstruct=True` 로 변환 Then 결합·구분자·헤딩 규약이 골든과 바이트 동일
      (비결정성을 프로바이더 경계 밖으로 격리 — 오케스트레이션/결합 로직만 회귀 측정).
- [ ] AC4 (실 VLM 제외): 실 VLM 호출 산출물은 비결정적이므로 **바이트 골든 대상에서 제외**하고,
      대신 **구조 불변식 어서션**(유효 Markdown, validator 경고 0, 표 정합, 슬라이드 수 보존)으로
      검증한다. 실 VLM 게이트를 CI hard-fail 로 둘지(비용/불안정성)는 `⟨DEC-6⟩`(ARCH 위임) 확정.
- [ ] AC5 (CI 정책): Given 모의 프로바이더 골든/라우팅 골든 중 하나라도 달라짐 When CI
      Then diff 출력 + 실패(머지 차단, FR-29 D-6 hard-fail 전례). 의도된 변경은 골든 갱신 PR.
- [ ] AC6 (경계·격리): Given 렌더/VLM 미가용 CI 환경 When 회귀 테스트 Then 렌더 의존 테스트는
      skip 처리(환경 게이트)되고 텍스트 경로·라우팅·모의 프로바이더 테스트는 항상 실행된다.
- [ ] AC7 (픽스처 자산): Given 회귀 스위트 When 공개 CI/배포 Then 골든·라우팅 픽스처는 전부
      **합성 PPTX**(사내 자료 미포함)이며 리포지토리·PyPI 산출에 사내 파일이 유입되지 않는다.

**우선순위**: Must / **의존**: FR-29(골든 게이트 확장 대상), FR-31·FR-33(측정 대상), FR-14(validator)

기존 v0.1.2 NFR 및 REQ-wave3 NFR 계승. 신규 하이브리드 경로에 적용되는 수치 기준:

- **NFR-V020-1 성능(렌더)**: 슬라이드 1장 렌더 서브프로세스(soffice)는 타임아웃 `⟨ARCH⟩`(기본 제안
  30s, vector.py 계승) 내 완료 또는 `None`. 텍스트 경로 슬라이드 어셈블 p95 는 기존 게이트
  (20슬라이드 < 5초, VLM 제외) 회귀 없음. 렌더/VLM 미개입 슬라이드에 추가 지연 0.
- **NFR-V020-2 비용(콜 수)**: 문서당 실제 VLM 콜 수 ≤ min(고유 복잡 슬라이드 수, `max_vlm_slides`).
  중복 렌더(sha256 동일)는 콜 1회로 합산(FR-34 AC3). 측정 가능한 테스트 존재.
- **NFR-V020-3 결정성/재현성**: 텍스트 경로 바이트 결정성 유지(INV-1). 라우팅 판정(FR-31)·결합
  순서(FR-33)는 동일 IR → 동일 결과. VLM 응답 본문만 비결정 — 회귀는 FR-35 로 흡수.
- **NFR-V020-4 하위호환**: 기존 `convert()`/`ConvertOptions()`/공개 API 무파손. 신규 옵션 default
  = 현행 출력 동등(opt-in). `describer=None`·재구성 프로바이더 `None` 시 현행 경로 그대로.
- **NFR-V020-5 격리/폴백**: 렌더/VLM 실패의 최대 영향 반경은 **해당 슬라이드 1장의 텍스트 폴백**.
  전체 변환·다른 슬라이드 무손상, 예외 미전파(INV-4). 부분 실패 문서도 유효 Markdown.
- **NFR-V020-6 의존 격리**: `assembler.py`·`validator.py` 최상단 VLM SDK·python-pptx·Pillow
  import 0 유지(INV-5). core-only 환경에서 `import pptx_md` 및 기본 `convert()` 성공.
- **NFR-V020-7 품질 게이트**: 신규 코드 라인 커버리지 ≥ 75%(프로파일 §7), mypy `src/` exit 0,
  `ruff check . && black --check .` exit 0, PII 로그 미출력(NFR-06).

## 10. Out of Scope (v0.2.0 배제 · Non-goals)

- **PowerPoint COM 렌더 신설** — DEC-1 에서 **기각 확정**(LibreOffice 채택). 신설 금지(과거 부채, ARCH-M12 §9.3 계승).
- **원본 대비 픽셀 완전 재현/전체 OCR 완전성** — VLM 재구성은 구조 보존이 목표이며 100% 문자
  전사·시각 완전 복원을 보장하지 않는다.
- **애니메이션·전환·발표 타이밍** 재현.
- **디스크/세션 간 렌더·재구성 캐시**(REQ P-01) — 별도 승인 영역(REQ-wave3 §6 계승).
- **사용자 정의 프롬프트 템플릿/heading 매핑 커스터마이즈** — v1 미요구(REQ-wave3 §6 계승).
- **폐쇄망/오프라인 로컬 VLM 번들링** — 프로파일 §2(공개 인터넷 가능) 전제, 오프라인 모델 미포함.
- **MkDocs 등 문서 사이트 렌더** — 산출물은 Markdown 문자열까지.
- **비시각(순수 텍스트) 슬라이드의 재구성** — 명시적으로 텍스트 경로 유지(비용 회피).
- **사내 실증 파일의 공개 픽스처화** — `.upload/SamsungSDS_AICC_제안서.pptx` 는 사내 자료로
  리포지토리/PyPI/CI 배포 산출에 미포함. 공개 회귀 픽스처는 합성 PPTX 로 대체(FR-35 AC7).

| FR | 기능 | 우선순위 | 직접 의존 |
