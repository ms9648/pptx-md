### 핵심 설계 메모 — "모의 프로바이더 골든"이 실 soffice 없이도 항상 실행되는 이유

`api.convert()` 는 렌더러/복잡도 함수 주입 지점을 공개하지 않는다(`ConvertOptions` 에 그런 필드가
없음 — ARCH-v020 §3.5 의도적 설계). `reconstruct_slides` 의 `renderer`/`complexity_fn` 은 키워드
전용 인자이므로 그 default 는 `__defaults__` 가 아니라 **가변** dict `__kwdefaults__` 에 저장된다
(파이썬 시맨틱 확인됨 — `reconstruct_slides.__kwdefaults__.keys()` 로 실증). 테스트는 이 dict 의
`"renderer"` 항목만 `monkeypatch.setitem` 하여 실제 공개 API `convert(..., visual_reconstruct=True,
reconstructor=Fake)` 호출 경로를 그대로 태우면서도 실 soffice/pypdfium2 를 전혀 건드리지 않는다.
`slide_render_available` 은 이름으로 임포트되어 있어(O 팀 설계 docstring 명시) 이것도 함께
monkeypatch 해 조기 return 을 우회한다. 결과적으로 soffice 가 전혀 설치되지 않은 이 검증 환경에서도
AC3 테스트는 스킵되지 않고 항상 실행된다(AC6 충족의 직접 근거, TC-96-23).
