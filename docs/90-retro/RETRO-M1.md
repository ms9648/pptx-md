## 3. 문제와 원인

### P1. 문서 산출물 커밋 지연
- **현상**: ARCH-M1.md, TR-8.md가 FR-01 PR(#10)에 포함되지 않고 미커밋 상태로 남아있다가 FR-02 PR(#11)에서 뒤늦게 커밋됨.
- **원인**: developer 산출물 저장 위치 규약(`docs/20-design/`, `docs/30-test/`)은 있으나, 해당 이슈 PR에 *즉시* 포함해야 한다는 명시 규약이 없었음.
- **영향**: docs 파일이 FR-02 PR에 섞여 이슈-산출물 추적성이 흐려짐.

### P2. 이슈 경계 초과 구현
- **현상**: FR-01(#8) 구현 PR에 FR-02 범위인 ci.yml이 함께 포함됨. 결과적으로 FR-02 이슈는 코드 변경 없이 문서 PR만 생성하는 형식적인 루프를 거쳤음.
- **원인**: ARCH-M1이 FR-01+FR-02를 하나의 WBS로 다뤘고, developer가 WBS 전체를 한 PR로 완성함. 이슈 경계 vs WBS 경계의 불일치.
- **영향**: 프로세스 오버헤드(FR-02용 PR/reviewer/tester 루프) 발생. 결과물 품질에는 영향 없음.

### P3. ci.yml 트리거 브랜치 명목 불일치
- **현상**: `push: branches: [main]`이나 실제 기본 브랜치는 `master`. master 직접 push 시 CI 미트리거.
- **원인**: ARCH-M1 §4.2 설계 초안이 `main`으로 작성됐고, developer가 그대로 구현. 프로파일에 브랜치명이 명시되지 않음.
- **영향**: 프로파일 §5(main 직접 push 금지)로 실제 운영 영향은 없으나, 설계-구현 명목 불일치. 후속 이슈로 추적 필요.

### P4. AC 경계 케이스 실증 불가
- **현상**: AC5(Python 3.10 거부)는 3.10 인터프리터 미설치로 SKIP. AC4(CI failure)는 실제 failure 런 증거 부재.
- **원인**: 환경 제약(단일 Python 버전) + 에이전트 안전 제약(의도적 위반 주입).
- **영향**: 대체 검증(정적 확인, 논리 검증)으로 수용. 실질 리스크는 낮음.
